# app03: メモ・ノートアプリ 作成計画

## Context

ユーザーが**app03: メモ・ノートアプリ**の作成を要求。app02（RPG風ToDoリスト）の構造をベースにしつつ、ノートアプリとしての機能を提供する。

**要件：**
- PWA対応（app02と同様）
- レスポンシブデザイン（PC・スマホ両対応）
- スワイプ操作対応（スマホ）

## 実装アプローチ

### 参考ファイル
- [app02/index.html](../app02/index.html) - HTML構造、PWA対応
- [app02/styles.css](../app02/styles.css) - レスポンシブCSS
- [app02/app.js](../app02/app.js) - クラスベース設計、LocalStorage管理
- [app02/manifest.json](../app02/manifest.json) - PWAマニフェスト
- [app02/sw.js](../app02/sw.js) - Service Worker

### 作成するファイル

```
app03/
├── index.html      # メインHTML
├── styles.css      # スタイルシート
├── app.js          # メインロジック
├── manifest.json   # PWAマニフェスト
└── sw.js          # Service Worker
```

## 機能仕様

### メイン機能
1. **メモの作成**
   - タイトル入力
   - 本文入力
   - カテゴリ選択（個人、仕事、アイデア、その他）
   - 作成日時の自動記録

2. **メモの一覧表示**
   - カード形式で表示
   - 作成日順にソート
   - カテゴリ別に色分け

3. **メモの編集・削除**
   - 既存メモをクリックで編集モード
   - 削除ボタンで削除
   - スマホ: スワイプ操作（右スワイプで編集、左スワイプで削除）

4. **検索機能**
   - タイトル・本文からキーワード検索
   - リアルタイムフィルタリング

5. **データ保存**
   - LocalStorageに全メモを保存
   - 自動保存機能

### PWA機能
- Service Workerによるオフライン対応
- ホーム画面への追加対応
- スタンドアロンモードで動作

### UI/UX
- レスポンシブデザイン
- カードベースのモダンなUI
- スムーズなアニメーション
- スマホでのスワイプ操作

## 実装手順

### 1. app03ディレクトリの作成
```bash
mkdir app03
```

### 2. index.htmlの作成
- ヘッダー（タイトル、説明）
- 検索バー
- メモ追加ボタン
- メモ一覧表示エリア
- メモ編集モーダル
- Service Worker登録スクリプト

### 3. styles.cssの作成
- app02をベースにしたレスポンシブデザイン
- カードベースのレイアウト
- カテゴリ別の色分け
- モーダルのスタイル
- スワイプアニメーション

### 4. app.jsの作成
- `NoteApp` クラスの定義
- メモのCRUD操作
- LocalStorageとの連携
- 検索機能
- スワイプ操作のハンドリング
- モーダルの制御

### 5. manifest.jsonの作成
- アプリ名: "Memo Notes"
- テーマカラー: ノートアプリらしい色（例: #3b82f6 青）
- アイコン設定

### 6. sw.jsの作成
- キャッシュ名: `memo-notes-v1`
- ファイルのキャッシュ戦略

## 技術スタック

- **HTML5**: セマンティックHTML
- **CSS3**: Flexbox, Grid, カスタムプロパティ
- **Vanilla JavaScript**: ES6+ クラス構文
- **LocalStorage**: データ永続化
- **Service Worker**: オフライン対応

## 検証方法

### 動作確認
1. `app03/index.html` をブラウザで開く
2. メモを追加できることを確認
3. メモを編集・削除できることを確認
4. 検索機能が動作することを確認
5. LocalStorageにデータが保存されることを確認（DevTools → Application → Local Storage）

### レスポンシブ確認
1. ブラウザのデベロッパーツールでデバイスエミュレーション
2. PCビュー（幅1024px以上）
3. タブレットビュー（幅768px）
4. スマホビュー（幅375px）

### PWA確認
1. Chrome DevTools → Application → Manifest
2. Service Workerが登録されていることを確認
3. オフラインモードで動作することを確認
4. 「アプリをインストール」プロンプトが表示されることを確認

### スワイプ操作確認（スマホ）
1. 実機またはタッチデバイスでアクセス
2. 右スワイプで編集モーダルが開くことを確認
3. 左スワイプで削除確認が表示されることを確認

## 備考

- app02の設計パターンを活用し、コードの一貫性を保つ
- カテゴリは最初は固定（個人、仕事、アイデア、その他）とし、将来の拡張性を考慮
- メモの内容は最大1000文字程度とする
- 検索は大文字小文字を区別しない
