## コンテキスト

**目的**: 1週間の現在地天気予報と服装アドバイスを画像付きで分かりやすく表示するアプリを作成する

**背景**: 既存のapp01-03はVanilla JavaScriptベースで構築されており、統一されたUIパターンとPWA対応が実装されている。app04ではこれらのパターンを継承しつつ、初めて外部API（OpenWeatherMap）を統合する。

## 推奨アプローチ

### 技術スタック
- **フロントエンド**: Vanilla JavaScript（既存パターン継承）
- **天気API**: OpenWeatherMap（無料枠：1,000,000回/月、日本語対応）
- **位置情報**: Geolocation API + 都市名入力（ハイブリッド）
- **PWA**: Service Worker + manifest.json（app03の実装を参照）

### アーキテクチャ

#### クラス構成
```javascript
class WeatherApp {
    constructor()
    async init()
    async getCurrentPositionWeather()
    async getWeatherByCity(cityName)
    async fetchWeatherData(lat, lon)
    renderTodayWeather(data)
    renderWeeklyForecast(data)
    renderClothingAdvice(temp, weatherCode, gender)
    toggleGender() // 男性/女性切り替え
    saveToCache(data)
    loadFromCache()
}
```

#### データフロー
1. ユーザーが位置情報を許可 → Geolocation APIで座標取得
2. 位置情報拒否時 → 都市名入力フォームを表示
3. OpenWeatherMap APIで天気データ取得
4. localStorageにキャッシュ保存（30分間有効）+ 性別設定保存
5. UIに今日の天気、週間予報、服装アドバイスを表示（性別に応じた服装）

### 服装アドバイスロジック

#### 性別切り替え機能
- 男性用服装ボタン（👨）と女性用服装ボタン（👩）を用意
- クリックで即座に服装アドバイスを切り替え
- 選択した性別をlocalStorageに保存

#### 男性の気温に基づく服装
| 気温 | 服装 | アイコン |
|------|------|----------|
| 25℃以上 | Tシャツ、半袖シャツ、短パン | 👕 |
| 20℃～24℃ | 長袖シャツ、薄手のパーカー | 👔 |
| 15℃～19℃ | 薄手のジャケット、カーディガン、ジーンズ | 🧥 |
| 10℃～14℃ | ジャケット、トレンチコート、長ズボン | 🧥 |
| 5℃～9℃ | 冬服、ダウンジャケット、厚手のズボン | 🧥 |
| 5℃未満 | 厚手のコート、マフラー、手袋、インナー | 🧣 |

#### 女性の気温に基づく服装
| 気温 | 服装 | アイコン |
|------|------|----------|
| 25℃以上 | ワンピース、半袖ブラウス、スカート | 👗 |
| 20℃～24℃ | 長袖ブラウス、薄手のカーディガン | 👚 |
| 15℃～19℃ | 薄手のジャケット、カーディガン、デニム | 🧥 |
| 10℃～14℃ | ジャケット、トレンチコート、ロングスカート/パンツ | 🧥 |
| 5℃～9℃ | 冬服、ダウンジャケット、厚手のボトムス | 🧥 |
| 5℃未満 | 厚手のコート、マフラー、手袋、タイツ、インナー | 🧣 |

#### 天候に基づく追加アドバイス（共通）
- 雨/雪: 🌂 傘、レインウェア、レインブーツ
- 強風: 🧣 風を通さない服、スカートの場合は注意
- 曇り: 🧥 薄手の上着を持参

## 実装ステップ

### Phase 1: 基盤構築
1. app04ディレクトリとファイル作成（index.html, app.js, styles.css）
2. 基本UI構造（ヘッダー、検索セクション、カードレイアウト）
3. CSS変数と既存アプリと一貫性のあるスタイル

### Phase 2: API連携
1. OpenWeatherMap APIキー取得と設定
2. 天気データ取得関数実装
3. Geolocation API実装
4. 都市名検索API実装
5. localStorageキャッシュ機能

### Phase 3: データ表示
1. 今日の天気カードの実装
2. 天気コードから絵文字アイコンへのマッピング
3. 週間予報グリッドの実装（5日分）
4. 日ごとの最高/最低気温計算

### Phase 4: 服装アドバイス
1. 男性/女性用服装データの実装
2. 性別切り替えボタンUIの実装
3. 気温ベースの服装判定ロジック実装（男女別）
4. 天候ベースのアクセサリー判定実装
5. 服装アドバイスカードの表示
6. localStorageへの性別設定保存

### Phase 5: PWA対応
1. manifest.json作成（app03を参照）
2. Service Worker実装（app03/sw.jsをベース）
3. オフライン対応とキャッシュ戦略

### Phase 6: 仕上げ
1. エラーハンドリング（APIエラー、ネットワークエラー）
2. ローディングアニメーション
3. 天気アイコンのCSSアニメーション
4. レスポンシブ対応調整
